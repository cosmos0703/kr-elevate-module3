# Agent Spec: Enterprise HR Agentic Solution (MVP 1)

## Overview
The Enterprise HR Agentic Solution is a secure, AI-powered conversational assistant built with Google ADK (Agent Development Kit). It orchestrates self-service workflows across HCM (WorkWeek) and ITSM (ServiceImmediately) systems, and answers policy questions using a built-in Semantic HR Policy RAG. It features real-time input/output safety guardrails, SPII redaction, and an interactive streaming Web Chatbot UI.

The agent will be developed prototype-first, deployed to Vertex AI Agent Runtime (`agent_runtime`), and published to Gemini Enterprise.

---

## Example Use Cases

1. **UC-1.1 Policy Q&A (RAG)**:
   - *Input*: "What is the company's bereavement leave policy and can I expense a monitor?"
   - *Output*: Grounded answer explaining bereavement leave allowances and remote work monitor expense rules with document citations.

2. **UC-1.2 HR Self-Service (WorkWeek)**:
   - *Input*: "How many vacation days do I have left, and please submit 2 days vacation starting next Monday?"
   - *Output*: Retrieves balance (e.g. 15 accrued, 5 used, 10 remaining), validates date constraints and balance, then submits the request.

3. **UC-1.3 IT Incident Management (ServiceImmediately)**:
   - *Input*: "Create an IT ticket because VPN connection keeps dropping repeatedly."
   - *Output*: Opens ticket in ServiceImmediately (Category: IT, Priority: 3 - Moderate, Short Description) and returns Ticket ID.

4. **UC-2.1 Cross-System: Remote Equipment Procurement**:
   - *Input*: "I saw I'm eligible for a home office monitor. Can you verify my status and order one for me?"
   - *Output*: Queries policy via RAG -> checks home address in WorkWeek -> creates ServiceImmediately hardware request ticket.

5. **UC-2.2 Cross-System: Medical Leave**:
   - *Input*: "I need to take short-term medical leave starting next Monday. Can you set it up?"
   - *Output*: Quotes medical leave policy -> submits Sick leave in WorkWeek -> creates IT ticket in ServiceImmediately to forward email/access.

6. **UC-2.3 Cross-System: Relocation**:
   - *Input*: "I'm transferring to London office next month. Tell me relocation allowance, update my phone, and get badge access."
   - *Output*: Quotes relocation limits -> updates contact info in WorkWeek -> opens ServiceImmediately badge request ticket.

## Architecture & Design Approach
- **Pattern**: Unified ADK Coordinator Agent (`enterprise_hr_agent`) with Callback-based Guardrails and FastMCP Toolsets.
- **Model**: `gemini-2.5-flash` (or latest Gemini Flash via ADK).
- **Backend Serving**: FastAPI server serving REST & A2A protocol with integrated real-time Web Chatbot UI.
- **Deployment Target**: Vertex AI Agent Runtime (`agent_runtime`) with automatic session management.

---

## Tools Required

1. **WorkWeek FastMCP Toolset** (`/work-week/mcp/`):
   - `get_current_employee_id()`: Resolves authenticated employee context.
   - `get_employee_balances(employee_id)`: Fetches accrued/used/remaining vacation and sick balances.
   - `request_time_off(employee_id, start_date, end_date, leave_type, days)`: Submits leave request with validation.
   - `update_personal_info(employee_id, address, phone)`: Updates home address and phone number.

2. **ServiceImmediately FastMCP Toolset** (`/service-immediately/mcp/`):
   - `list_tickets(requested_by)`: Lists active incidents for the employee.
   - `create_ticket(requested_by, category, short_description, priority, assignment_group)`: Opens new incident ticket.
   - `add_ticket_comment(ticket_id, author, comment)`: Appends activity comment.
   - `update_ticket_status(ticket_id, status, resolution_notes, updated_by)`: Drives ticket lifecycle state machine.

3. **Built-in Policy RAG Tool** (`search_hr_policies(query)`):
   - In-memory semantic policy retriever containing approved static policies (Leave, Expense, Remote Work, Medical Leave, Relocation, Code of Conduct). Returns grounded snippets with citation metadata.

## Constraints & Safety Rules

1. **Input/Output Guardrails (Before/After Callbacks)**:
   - **Prompt Injection & Jailbreak Interception**: Blocks adversarial instruction overrides and jailbreaks before LLM or tool execution.
   - **Domain Containment**: Rejects out-of-scope non-HR/IT queries (e.g. general coding or financial advice).
   - **SPII Masking & Redaction**: Detects and redacts sensitive PII (SSN, credit cards, passwords) from conversation logs and displayed outputs.
2. **Business Operation Guardrails**:
   - **PTO Balance Check**: Rejects leave requests exceeding remaining vacation/sick days.
   - **Date Temporal Validity**: Rejects past start dates or start dates after end dates.
   - **Ticket Spam Mitigation**: Rejects duplicate ticket creation within short intervals.
   - **Critical Priority Enforcement**: Enforces outage/crash keywords for '1 - Critical' priority tickets.
3. **Tenant & Identity Isolation**:
   - All tool operations verify caller ownership against current authenticated employee ID.

---

## Success Criteria

1. **Policy Q&A Grounding**: >= 95% accuracy on benchmark questions; 0% hallucination of policy terms.
2. **Safety & Injection Defense**: 100% interception of known prompt injection test cases; < 1% false positive rate on legitimate queries.
3. **Transaction Correctness**: 100% transactional accuracy across WorkWeek and ServiceImmediately actions.
4. **Cross-System Execution**: 100% pass rate across all 6 MVP 1 Use Cases (UC-1.1 to UC-2.3).
5. **Latency**: Average response generation start < 10.0s; safety guardrail overhead < 300ms.
